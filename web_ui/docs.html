Full API Reference

This server exposes REST endpoints used by the web UI for authentication, cluster orchestration, status streaming, control actions, and model export. The canonical machine-readable schema is available at /api/openapi.json, and interactive execution is available at /docs/swagger.

Best-practice usage: call GET /api/auth/mode first, authenticate if required, then call protected APIs with the nm_session cookie on the same origin.

Authentication Flow

1. Discover mode via GET /api/auth/mode. 2. If mode=local, call POST /api/login with username/password. 3. If mode=oidc, use browser redirect via GET /auth/oidc/login. 4. Session cookie nm_session is then used for protected APIs. 

curl -s http://localhost:8080/api/auth/mode
curl -s -X POST http://localhost:8080/api/login \
  -H 'content-type: application/json' \
  -d '{"username":"admin","password":"change-me"}' -i
curl -s http://localhost:8080/api/status --cookie "nm_session=<value>"

Interactive Usage

Use Swagger UI to test requests directly in-browser. It is preconfigured for same-origin cookies so you can login and invoke protected endpoints in one session.

If OIDC mode is enabled, complete browser login first, then open Swagger UI to run authenticated API calls.
